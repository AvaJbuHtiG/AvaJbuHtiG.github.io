    安装

    habse运行需要hdfs和zookeeper的支持，所以安装前需要保证hdfs和zookeeper正常。

    配置文件

    主要修改3个配置文件

    hbase-env.sh

    hbase-site.xml

    主要是配置hbase在hdfs上的路径，hbase的端口，zookeeper的信息

    3、regionServer

    配置分区服务器的主机

    分发到各个节点，启动 start-habse.sh

    hbase shell

    list

    列出当前表

    创建表

    create table_name […列族名]

    插入数据

    put ‘table_name’, ‘rowkey’, ‘列族名:列名’, ‘数据’

    表示向student表的info列族的sex列，插入值 ‘male’,其中rowkey类似于主键，唯一的列标志，可以确定唯一的列

    更新数据

    put ‘table_name’, ‘rowkey’, ‘列族名:列名’, ‘数据’

    和插入数据一样，指定rowkey一样，就会更新数据

    删除数据

    删除某个rowkey的全部数据

    deleteall ‘table_name’, ‘roekey’

    删除student表中，rowkey为10001的一行

    查看表结构

    describe table_name

    查看指定行数据

    get ‘table_name’, ‘rowkey’ 查看某一行数据

    get ‘table_name’, ‘rowkey’, ‘列族名:列名’ 查看某一行的某一列数据

    统计表数据行数

    count ‘table_name’

    清空表数据

    truncate ‘table_name’

    删除表

    删除表需要先禁用表，然后再删除

    disable ‘table_name’

    drop ‘table_name’

    变更表信息

    alter ‘table_name’

    扫描表数据

    scan ‘table_name’ , {startrow => ‘起始rowkey’, stoprow => ‘结束rowkey’} stoprow 可缺省

    hbase读写数据流程

    读取数据流程

    首先要知道，hbase的所有表的原数据都是存在habse:meta表中的，这里面记录了所有表数据的位置信息

    1、habse先访问zookeeper，获取neta表的region信息

    2、通过meta表所在的region信息，请求region服务器，查询meta表，根据namespace，表名，rowkey在meta表中找到目标表对应的region信息

    3、请求目标表所在region，查找对应的region，先会从MemStore找数据，如果没有，再到BlockCache中读，BLockCache还没有，再到StoreFile上读（从StoreFile中读取的数据，会向BlockCache中读取一份）。

    4、获取到的数据写会客户端（如果是从StoreFile里读取的数据，不是直接返回给客户端，而是会先写入BlockCache，在返回给客户端）

    写入数据流程

    1、client先查询目标表所在的region信息，然后向HregionServer发送写请求

    2、HregionServer将数据写到HLog(为了数据的持久化和恢复)

    3、HregionServer将数据写到内存(MemStore)

    4、响应client，写入成功

    数据flush

    当MemStore数据达到阈值(默认128M)，会将数据写到磁盘，将MemStore中数据删除，同时删除Hlog中对应的历史数据。

    然后将溢写出来的数据存放到hdfs中，同时在Hlog中做标记点

    数据合并

    再这样一种情况下，(某个regionServer上的，region中的MemStore不够分配到128M内存是，也会触发数据的flush)，当flush出来的数据块达到3块数，Hmaster会触发合并操作。

    Region将数据块加载到本都，进行合并，如果合并的数据超过了256兆，会将数据进行拆分，将拆分后的分配给不同的HregionServer(表的某个列族多了一个分区)

    预分区

    hbase中，每一个region维护着startRow与endRowKey，如果加入的数据符合某个region维护的rowKey范围，则该数据交给这个region维护。当region中数据量达到一定范围时，就会自动拆分，所以，如果预先分好区，可以提高hbase的性能

    预分区方式介绍一下几种

    1、手动设定预分区

    该表初始就会有5个分区

    2、生成16进制序列预分区

    这个表的分区键是从 11111111 - eeeeeeee的字符串

    3、按照自己指定的分区键来分区

    在splits.txt文件中写入自己指定的分区键，该文件需要放在habse的根目录下

    4、在用api创建表时，自己定义分区键
